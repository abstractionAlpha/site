Minecraft Manhunt

Created: October 16 2021

Modified: October 16 2021

Open Source: Yes

Summary

Minecraft Manhunt is a Minecraft (Java edition) server plugin that is meant to replicate the Minecraft Manhunt series by WilburSoot, revolutionized by Dream. This plugin uses multi-threading and basic object-oriented design in order to create the gamemode.

Commands:

  • manhunt: Loads the GUI for Minecraft Manhunt. See more about usage below.
Permissions:
  • manhunt.main: Allows use of the manhunt command.

Usage

Workflows

The main and only workflow of this plugin is with the manhunt command.

On executing the command, given appropriate permissions are had, the player who executed it will be presented with a GUI. This GUI can only be accessed by one player at a time, and cannot be accessed once the game has started. The GUI has three components:

  • A Runners button, implemented as a player head
  • A Hunters button, implemented as a player head
  • A Confirm button, implemented as a piece of stained clay

The Runners button opens a separate GUI that contains a whole bunch of player heads. Each player head in this tab is the head of a runner. The runners are stored in an array sorted by name. If you left-click on one of the players, they will be moved from this list to the Hunters list. If this list is empty, it will just show a steve. If you click the steve head, then it will open the Hunters list.

The Hunters button opens a separate GUI that contains a bunch of player heads. Each player head in this tabe is the head of a hunter. The hunters are stored in an array sorted by name. If you left-click on one of the players, they wil be moved from this list to the Runners list. If this list is empty, it will just show a steve. If you click the steve head, then it will open the Runners list.

The Confirm button is a piece of stained clay. It is red if either of the two lists are empty, and green otherwise (until later pressed). If the clay is red then pressing the button doesn't do anything. If the clay is green then pressing the button will turn the clay blue. If the clay is blue then pressing the button will start the game. (See Gameplay section)

Gameplay

After going through the main workflow, the game begins! All players can play Minecraft as normal (unless friendly fire is turned off in the config). If all of the hunters die, then the game ends.

The location of each runner is held in a backing store, simply in the form of an object represnting the runner. The runners' positions will automatically be updated whenever any runner goes through any portal. The runners' positions can be manually updated if a hunter right-clicks their tracking compass. The hunters' compasses update automatically when they go through a portal, as well as manually when they right-click said compass.

When any runner dies, all players will recieve a notification. If both runners die, the game ends and all of the dynamic objects are destroyed for a fresh run.
